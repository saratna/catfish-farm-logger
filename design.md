# Catfish Farm Logger 設計文書

## 設計方針

本アプリは、フィリピンのナマズ養殖現場で、通信が不安定な環境でも日々の検査、給餌、体重、写真記録を確実に残すことを目的とする。基本設計は**オフラインファースト**とし、入力したデータは端末内に先に保存され、インターネット接続が回復した時点でGoogle Driveへ同期する。利用状況は屋外、片手操作、短時間入力を想定し、画面は縦向き 9:16、親指が届きやすい下部操作を重視する。

Apple Human Interface Guidelines に沿い、情報を詰め込みすぎず、状態が一目で分かるカード型UIを採用する。検査漏れや同期未完了は、色と短い文言で明確に示す。現場での誤入力を減らすため、数値入力、選択肢、保存完了フィードバックを明確にする。

## 画面一覧

| 画面 | 主な内容 | 主な操作 |
|---|---|---|
| ホーム | 本日の検査状況、同期状態、水槽一覧、未完了タスク | 水槽選択、日次検査開始、同期実行 |
| 水槽詳細 | 水槽ごとの最新水温、検査履歴、給餌履歴、写真数 | 検査追加、給餌追加、写真追加 |
| 日次検査フォーム | 水温、pH、溶存酸素、アンモニア、亜硝酸、塩分またはメモ | 検査値保存、未入力警告確認 |
| 給餌・体重フォーム | 餌タイプ、餌量、平均体重、個体数メモ、観察メモ | 給餌記録保存 |
| 写真記録 | 水槽・ナマズ写真、撮影日時、説明メモ | カメラ起動、写真保存 |
| 同期設定 | Google Drive同期状態、水槽別フォルダ名、最終同期日時 | 同期実行、エクスポート確認 |
| 設定 | 検査項目、通知時刻、餌タイプ候補、単位 | 基本設定変更 |

## 主要コンテンツと機能

ホーム画面では、本日検査が終わっていない水槽を上部に表示する。通信状態は「オフライン保存中」「同期待ち」「同期済み」の三段階で表示し、現場担当者がデータの安全性を直感的に把握できるようにする。水槽一覧はカード形式とし、水槽名、今日の検査完了状況、最新水温、最後の給餌時刻を表示する。

水槽詳細画面では、特定の水槽に紐づく全記録へアクセスする。検査、給餌、写真の追加ボタンは画面下部にまとめ、片手で操作しやすくする。履歴は最新順で表示し、重要値が危険域に近い場合は警告色で示す。

日次検査フォームでは、最低限の必須項目として**水温**を設定し、その他の検査項目として pH、溶存酸素、アンモニア、亜硝酸などを入力できるようにする。現場によって検査体制が異なるため、検査項目は設定画面で表示・非表示を調整できる設計にする。

給餌・体重フォームでは、餌タイプ、餌量、ナマズの重さを記録する。餌タイプはあらかじめ登録した候補から選べるようにし、入力のばらつきを減らす。体重は平均体重として扱い、測定方法のメモも任意で残せるようにする。

写真記録では、端末カメラまたは写真ライブラリから画像を追加し、水槽ID、撮影日時、メモを紐づけてローカル保存する。Google Drive同期時には、水槽ごとのディレクトリ配下に写真とJSONデータをまとめる設計とする。

## 主要ユーザーフロー

| フロー | 手順 | 完了状態 |
|---|---|---|
| 初回利用 | アプリ起動 → 水槽を追加 → 通知時刻を確認 → ホームへ戻る | 水槽カードが表示される |
| 日次検査 | ホームで未検査水槽をタップ → 日次検査フォーム → 水温と検査値入力 → 保存 | ホームの本日検査が完了表示になる |
| 給餌記録 | 水槽詳細 → 給餌追加 → 餌タイプ・餌量・体重入力 → 保存 | 給餌履歴に最新記録が追加される |
| 写真記録 | 水槽詳細 → 写真追加 → カメラまたはライブラリ選択 → メモ入力 → 保存 | 写真が水槽に紐づく |
| 同期 | 同期画面 → 接続状態確認 → Google Driveへ同期 → 結果確認 | 同期待ち件数が減る |

## 色選定

ブランドカラーは、養殖池の水と健康管理を想起させる**ディープティール**を中心にする。背景は日中の屋外でも読みやすい淡い水色系、警告は検査漏れが直感的に分かるアンバー、危険値やエラーは落ち着いた赤を用いる。

| 用途 | 色 | 役割 |
|---|---:|---|
| Primary | `#087F8C` | 主要ボタン、選択状態、同期実行 |
| Background | `#F3FBFA` | 画面背景 |
| Surface | `#FFFFFF` | カード、フォーム |
| Foreground | `#102A2C` | 主要テキスト |
| Muted | `#607477` | 補助テキスト |
| Success | `#2E8B57` | 検査完了、同期済み |
| Warning | `#D98E04` | 検査未完了、同期待ち |
| Error | `#C2413B` | 入力エラー、同期失敗 |

## データ保存とGoogle Drive同期設計

ローカル保存は、端末内の永続ストレージにJSON形式で保持する。水槽、検査、給餌、写真メタデータ、同期状態を分離し、通信がなくても画面表示と記録追加が完結するようにする。Google Drive同期は、まずアプリ内のエクスポートファイルを作成し、接続時にアップロードできる構造を採用する。水槽ごとにDrive上のディレクトリを分け、各ディレクトリに検査データ、給餌データ、写真を格納する。

| データ | ローカル保存内容 | Drive上の想定配置 |
|---|---|---|
| 水槽 | ID、名前、場所、作成日時 | `/CatfishFarmLogger/{tankName}/tank.json` |
| 日次検査 | 水槽ID、日時、水温、検査項目、メモ | `/CatfishFarmLogger/{tankName}/inspections.json` |
| 給餌 | 水槽ID、日時、餌タイプ、餌量、体重、メモ | `/CatfishFarmLogger/{tankName}/feedings.json` |
| 写真 | 水槽ID、画像URI、日時、メモ | `/CatfishFarmLogger/{tankName}/photos/` |
| 同期状態 | 最終同期日時、同期待ち件数、失敗理由 | `/CatfishFarmLogger/sync-log.json` |

## 通知と検査促進

日次検査は、各水槽について一日一回の記録を促す。通知はローカル通知として扱い、通信に依存しない。ホーム画面では、通知に加えて未完了カードを表示し、アプリを開いた時点で検査漏れが見えるようにする。

## 実装上の前提

Google Driveへの直接自動アップロードは、Androidアプリ側でGoogle認証とDrive API連携が必要になる。初期版では、オフライン記録、同期対象データのローカル生成、Google Drive同期画面、同期状態管理を実装し、認証情報が利用可能になった段階でDrive APIの実アップロードに接続できる構造にする。ユーザーに認証情報を通常チャットで尋ねず、必要になった段階で安全な入力方法を用いる。

## 天気予報・飼育リスク・給餌助言の追加設計

この追加機能は、屋外の養殖池で片手操作しながら、現在地、気象変化、日々の給餌、魚の観察を短時間で確認できることを重視する。天気予報はナマズの水槽・池そのものの水質を直接測定するものではないため、画面上では**「診断」ではなく「確認を促すリスク予測」**として扱う。高温、強雨、気圧急変、湿度の高さなどが見えた場合、アプリは溶存酸素、水温、pH、アンモニア、亜硝酸などの現場測定を促す。

| 追加画面 | 主な内容 | 主な操作 |
|---|---|---|
| 天気・リスクダッシュボード | GPS位置、現在の気温、湿度、気圧、雨量、予報、リスクカード | 現在地取得、天気更新、アラート確認 |
| 気象記録詳細 | 取得元、取得時刻、座標、気象値、予報差分 | 記録確認、日次検査との紐づけ |
| 給餌アドバイス | 餌の商品名、タンパク質率、粒径、浮上/沈降、製造日、給餌量、魚体重 | 餌登録、推奨量確認、注意点確認 |
| アラート詳細 | リスク理由、根拠、推奨確認項目、次に記録すべき水質項目 | 確認済みにする、検査フォームへ進む |

ホーム画面には、従来の検査状況に加えて「本日の天気リスク」カードを上部に追加する。このカードは、通常時はティール、注意時はアンバー、危険時は赤で表示し、文言は短く「高温で溶存酸素低下に注意」「強雨後はpHと濁りを確認」など、現場ですぐ行動できる内容にする。カードをタップすると、天気・リスクダッシュボードへ遷移する。

給餌フォームは、従来の餌タイプと餌量だけでなく、**商品名、タンパク質率、浮上/沈降、粒径、製造日または開封日、残餌の有無、食いつき**を記録できる構成に拡張する。推奨給餌量は、魚体重、個体数、気温または水温、天気リスク、直近の食いつきを組み合わせ、過剰給餌による水質悪化を避ける方向で提示する。

| 追加フロー | 手順 | 完了状態 |
|---|---|---|
| 天気取得 | ホーム → 天気カード → 位置情報許可 → 天気更新 | 位置・気象値・リスクが保存される |
| リスク確認 | 天気カードまたは通知 → アラート詳細 → 推奨測定項目確認 | アラートが確認済みになる |
| 給餌助言 | 水槽詳細 → 給餌追加 → 餌商品と魚体重入力 → 推奨量確認 → 保存 | 給餌履歴と助言が保存される |
| 強雨後確認 | アラート詳細 → 日次検査フォーム → pH・溶存酸素・濁りメモ入力 | 気象リスクと検査記録が紐づく |

データ保存には、気象観測、予報、リスク判定、餌商品情報を追加する。初期実装では端末内のローカル保存を優先し、将来のGoogle Drive同期では既存の水槽別ディレクトリ配下に追記できるJSON構造を採用する。

| 追加データ | ローカル保存内容 | Drive上の想定配置 |
|---|---|---|
| 地点 | 緯度、経度、取得日時、手動/自動区分 | `/CatfishFarmLogger/location.json` |
| 気象記録 | 取得元、日時、気温、湿度、気圧、雨量、風、予報 | `/CatfishFarmLogger/weather.json` |
| リスク判定 | 対象日時、リスク種別、重要度、理由、推奨行動、確認済み | `/CatfishFarmLogger/risk-alerts.json` |
| 餌商品 | 商品名、タンパク質率、粒径、浮上/沈降、製造日、メモ | `/CatfishFarmLogger/feed-products.json` |
| 給餌助言 | 推奨量、算出理由、気象条件、魚体重、残餌・食いつき | `/CatfishFarmLogger/{tankName}/feeding-advice.json` |

## 成長計測・写真診断の追加設計

成長計測画面は、手入力の体長・体重記録を中心にしつつ、写真からの参考推定と外観異常チェックを同じ画面で扱う。屋外で魚を保持しながら操作する場面を想定し、画面上部には直近の成長状態、中央には入力フォーム、下部には「写真を撮る」「写真を選ぶ」「保存」の主要操作を配置する。写真診断は病名を断定せず、**外観上の注意サインを見つける補助機能**として明示する。

| 追加画面 | 主な内容 | 主な操作 |
|---|---|---|
| 成長・写真診断 | 直近の平均体長・体重、成長状態、写真チェック結果、履歴 | 体長・体重入力、写真撮影、写真選択、AIチェック、保存 |
| 成長履歴詳細 | 測定日、体長、体重、推定値、メモ、写真サムネイル | 履歴確認、異常値確認 |
| 写真チェック結果 | 推定体長・推定体重、見える外観サイン、信頼度、確認推奨項目 | 結果確認、測定記録へ反映、専門家相談メモ |

成長判定は、直近2点以上の測定履歴から体重ベースの増加率を計算し、「順調」「やや停滞」「急変注意」「データ不足」のような現場向けの短い状態文に変換する。MSU Extension が示すように、給餌・水質・魚体サイズ・飼育方式が成長に影響するため、判定は標準値との厳密比較ではなく、**同じ水槽内の履歴変化を主軸にした傾向表示**とする。

写真チェックでは、UF/IFAS Extension と SRAC が示す魚病の初期観察項目に合わせ、赤み・出血斑、潰瘍、腹部膨満、眼の突出、ヒレの損傷、白点または綿状付着、体色の異常、痩せまたは極端な膨らみを確認する。結果カードには「水質測定を確認」「食いつき低下が続く場合は隔離・専門家相談」など、次の行動を短く表示する。

| 追加データ | ローカル保存内容 | Drive上の想定配置 |
|---|---|---|
| 成長測定 | ID、水槽ID、測定日時、体長cm、体重g、写真URI、メモ、作成日時 | `/CatfishFarmLogger/{tankName}/growth-measurements.json` |
| 写真AIチェック | 測定ID、画像URI、推定体長cm、推定体重g、外観サイン、信頼度、推奨行動、注意書き | `/CatfishFarmLogger/{tankName}/photo-assessments.json` |
| 成長判定 | 対象水槽、対象期間、体重増加率、体長増加、判定、理由 | `/CatfishFarmLogger/{tankName}/growth-status.json` |

| 追加フロー | 手順 | 完了状態 |
|---|---|---|
| 手入力で成長記録 | 成長タブ → 水槽選択 → 体長・体重入力 → メモ確認 → 保存 | 成長履歴に測定値が追加され、状態が再計算される |
| 写真から参考推定 | 成長タブ → 写真を撮る/選ぶ → AIチェック → 推定値を確認 → 必要なら手入力で補正 → 保存 | 写真URIとチェック結果が測定記録に紐づく |
| 異常サイン確認 | 写真チェック結果 → 赤み・潰瘍・膨満などの表示確認 → 水質測定または専門家相談メモ | 記録に注意サインと推奨行動が残る |

## 世界知見・養殖経営管理の追加設計

この追加機能では、秋篠宮皇嗣殿下の東南アジア産ナマズ研究、フィリピン政府機関、SEAFDEC/AQD、FAO、USDA ARS、Mississippi State University Extension などの一次資料・公的資料を、アプリ内の**出典付き知見カード**として整理する。知見カードは、病気・成長・種苗・給餌・経営の各画面で、現場判断の補助として短く表示する。特定の病名や収益性を断定せず、記録されたデータと出典に基づき「次に確認すべきこと」を提示する。

| 追加画面 | 主な内容 | 主な操作 |
|---|---|---|
| 知見ライブラリ | 秋篠宮皇嗣殿下関連研究、BFAR、PCAARRD、SEAFDEC、FAO、USDA、MSUの要点と出典 | 分野別フィルタ、出典確認、関連記録画面へ移動 |
| 収支ダッシュボード | 総費用、売上、粗利益、利益率、kgあたり費用、kgあたり利益 | 期間切替、費用追加、販売追加 |
| 費用入力 | 稚魚代、餌代、人件費、電気代、水道代、薬剤費、設備費、その他費用 | 水槽選択、金額・数量・単価入力、保存 |
| 販売入力 | 販売日、販売重量kg、販売尾数、販売単価、販売先、メモ | 売上記録保存、粗利益再計算 |
| 収支履歴 | 月別・水槽別の費用と売上、利益推移 | 履歴確認、記録編集の入口 |

収支ダッシュボードは、屋外の現場でも一目で状況が分かるように、上部に「今月の粗利益」「kgあたり費用」「kgあたり利益」を大きく表示する。費用と販売は別々に入力できるが、同じ水槽IDと期間で集計し、成長測定や給餌履歴と突き合わせられる構造にする。価格変動が重要課題であるため、販売単価の履歴を残し、同じ出荷重量でも利益が変動することを見える化する。

| 追加データ | ローカル保存内容 | Drive上の想定配置 |
|---|---|---|
| 知見カード | ID、分類、題名、要点、出典名、URL、適用画面、注意書き | `/CatfishFarmLogger/knowledge-library.json` |
| 費用記録 | ID、水槽ID、日付、費目、数量、単位、単価、金額、通貨、メモ | `/CatfishFarmLogger/{tankName}/costs.json` |
| 販売記録 | ID、水槽ID、日付、販売重量kg、販売尾数、販売単価、売上、販売先、メモ | `/CatfishFarmLogger/{tankName}/sales.json` |
| 収支集計 | 対象期間、水槽ID、総費用、売上、粗利益、利益率、kgあたり費用、kgあたり利益 | `/CatfishFarmLogger/{tankName}/profit-summary.json` |

| 追加フロー | 手順 | 完了状態 |
|---|---|---|
| 費用記録 | 収支タブ → 費用追加 → 費目選択 → 数量・単価または金額入力 → 保存 | 費用履歴と収支ダッシュボードが更新される |
| 販売記録 | 収支タブ → 販売追加 → 重量・尾数・単価入力 → 保存 | 売上と粗利益が再計算される |
| 収支確認 | 収支タブ → 期間選択 → 総費用・売上・利益確認 | 期間別の採算性が分かる |
| 知見確認 | ホームまたは知見タブ → 分野を選択 → 出典付き要点を確認 | 関連する記録項目の確認・入力へ進める |

## FCR・利益率連動アラート追加設計

### 目的

ナマズ養殖では、給餌量が増えていても魚体重が十分に増えない場合、餌代の増加が粗利益率を直接圧迫します。本機能では、**推定FCR（投入飼料kg ÷ 増加バイオマスkg）**、**粗利益率**、**餌代比率**を同じ水槽・ロット単位で連動評価し、飼育者が早めに給餌、水質、疾病、販売単価、固定費の見直しを行えるようにします。

| 画面 | 追加内容 | 主な表示 |
|---|---|---|
| 収支ダッシュボード | 経営アラートカード | 推定FCR、増加バイオマス、総給餌量、利益率、餌代比率、警告レベル |
| 収支ダッシュボード | アラート詳細リスト | FCR悪化、低利益率、餌代比率上昇、FCR悪化と低利益率の同時発生 |
| Drive Export | 経営アラート出力 | 水槽別JSONにアラート評価結果を保存 |

### 判定の考え方

FCRは既存の給餌記録、成長測定、尾数入力から推定します。複数回の平均体重記録がない、または尾数が未入力の場合は無理に数値を作らず、**FCR未計算アラート**として表示します。粗利益率は既存の販売額と総コストから算出し、販売記録がない場合は未計算として扱います。これにより、架空の標準値を作らず、入力済みデータだけを根拠にした判定に限定します。

| 判定対象 | 注意条件 | 危険条件 | 推奨行動 |
|---|---|---|---|
| 推定FCR | 2.2以上 | 3.0以上 | 残餌、食い付き、粒径、水質、疾病サインを確認 |
| 粗利益率 | 15%未満 | 0%未満 | 販売単価、餌代、人件費、電気・水道費を見直す |
| 餌代比率 | 55%以上 | 70%以上 | 餌銘柄、給餌率、保存状態、残餌を確認 |
| 連動リスク | FCR 2.2以上かつ利益率15%未満 | 同時発生を危険表示 | 給餌抑制、水質測定、販売条件確認を同日に行う |

### 表示上の注意

アラートは会計上の確定診断ではなく、現場記録に基づく**早期確認サイン**として扱います。死亡数、選別、未記録給餌、在池尾数の誤差、平均体重サンプルの偏りによりFCR推定は変動するため、画面内に限界を明示し、最終判断は実測と帳票確認で行うよう促します。


## 収支インサイト拡張設計（2026-05-04）

収支画面には、選択水槽の月次FCR・粗利益率推移、全水槽を横断した採算ランキング、アラート内容に連動する改善チェックリストを追加する。月次グラフは片手操作の縦長画面で読めるようにカード内へ簡易SVG折れ線として表示し、詳細な月別値は直近3か月の行で補足する。ランキングは選択タブの影響を受けず、農場全体の水槽を利益率、粗利益、名称の順で並べる。改善チェックリストは危険・注意アラートが存在する場合だけ表示し、チェック状態は画面内の一時状態として保持する。Google Driveエクスポートには全体の月次推移とランキング、各水槽の月次推移と改善チェックリストを含める。
